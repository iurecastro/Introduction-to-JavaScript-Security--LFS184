Supply Chain Defense Strategies

Practical protection techniques against dependency attacks

1. Use Lockfiles and Precise Versions

Lockfiles ensure reproducible builds and prevent unexpected dependency changes.

Commit package-lock.json to version control
Use exact versions in package.json (avoid ^ or ~)
Enable CI checks for lockfile consistency
Regularly update dependencies with audit
// package.json - Use exact versions { "dependencies": { "express": "4.18.2", // Exact version "lodash": "4.17.21" // Not "^4.17.21" } } // package-lock.json ensures reproducibility // Always commit this file!

  • Prevents "dependency drift"
  • Ensures all environments are identical
  • Mitigates dependency confusion attacks

  • Larger version control diffs

2. Minimize Dependencies

Reduce attack surface by minimizing both direct and transitive dependencies.

Regularly audit and remove unused dependencies
Prefer smaller, focused packages
Evaluate dependency maintenance status
Consider alternative implementations
üîß Using depcheck to find unused dependencies:
# Install depcheck npm install -g depcheck # Check for unused dependencies depcheck # Output shows unused dependencies: Unused dependencies: * axios * moment * underscore Unused devDependencies: * mocha * chai

3. Use Scanning and Monitoring Tools

Automated tools can detect vulnerabilities and suspicious behavior.

Integrate scanning into CI/CD pipelines
Use both CVE-based and behavioral analysis
Monitor for package behavior changes
Set up automated security alerts
# GitHub Actions workflow for security scanning name: Security Scan on: [push, pull_request] jobs: security-scan: runs-on: ubuntu-latest steps: - uses: actions/checkout@v4 - uses: actions/setup-node@v4 - name: Install dependencies run: npm ci - name: Run npm audit run: npm audit - name: Run Socket scan run: npx socket-cli@latest analyze . - name: Check for typosquats uses: socketsecurity/github-action@main

4. Implement Automation

Automate security checks to catch issues early.

Pre-commit hooks for dependency validation
CI/CD pipeline security gates
Automated dependency updates (Dependabot)
Deployment approval workflows

5. Verification and Signing

Verify package integrity and authenticity.

Use package signing (Sigstore, GPG)
Verify checksums and signatures
Implement artifact provenance
Use SBOM (Software Bill of Materials)
# Verifying package integrity with npm npm install --verify # Checking package signatures npm audit signatures # Generating SBOM npx @cyclonedx/cyclonedx-npm --output-file sbom.json

Security is a team responsibility.

Regular security training sessions
Clear dependency adoption policies
Incident response plans for supply chain attacks
Security champions program
