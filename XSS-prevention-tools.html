Content Security Policy (CSP)

CSP is a critical security layer that restricts sources of scripts, styles, and other resources.

Strict CSP Configuration:

// HTTP Header Content-Security-Policy: default-src 'self'; script-src 'self'; style-src 'self' 'unsafe-inline'; img-src 'self' data: https:; font-src 'self'; connect-src 'self'; frame-ancestors 'none'; base-uri 'self'; form-action 'self'; object-src 'none'; script-src-attr 'none'; upgrade-insecure-requests;

Implementing CSP in Express.js:

const express = require('express'); const helmet = require('helmet'); const app = express(); // Basic CSP with Helmet app.use(helmet.contentSecurityPolicy({ directives: { defaultSrc: ["'self'"], scriptSrc: [ "'self'", "'sha256-abc123...'" // Hash of inline script ], styleSrc: ["'self'", "'unsafe-inline'"], imgSrc: ["'self'", "data:", "https:"], fontSrc: ["'self'"], connectSrc: ["'self'"], frameSrc: ["'none'"], objectSrc: ["'none'"], upgradeInsecureRequests: [], }, reportOnly: false // Enforce, don't just report })); // For development, use report-only mode first app.use(helmet.contentSecurityPolicy({ directives: { defaultSrc: ["'self'"], scriptSrc: ["'self'"], // ... other directives }, reportOnly: true, // Only report violations reportUri: '/csp-violation-report' })); app.post('/csp-violation-report', (req, res) => { console.log('CSP Violation:', req.body); res.status(204).end(); });
üîç CSP Benefits:
  • Prevents inline script execution
  • Blocks eval() and similar functions
  • Restricts external script sources
  • Prevents clickjacking with frame-ancestors
  • Helps detect XSS attempts via violation reports

Modern Framework Protection

React XSS Protection

React automatically escapes content in JSX, but there are still risks:

// SAFE - React auto-escapes function Welcome({ name }) { return <div>Hello, {name}</div>; // If name = '<script>alert(1)</script>' // Output: Hello, &lt;script&gt;alert(1)&lt;/script&gt; } // DANGEROUS - dangerouslySetInnerHTML function Dangerous({ html }) { return <div dangerouslySetInnerHTML={{ __html: html }} />; } // SAFE - With sanitization import DOMPurify from 'dompurify'; function SafeHTML({ html }) { const cleanHTML = DOMPurify.sanitize(html, { ALLOWED_TAGS: ['b', 'i', 'em', 'strong', 'a'], ALLOWED_ATTR: ['href', 'title'] }); return <div dangerouslySetInnerHTML={{ __html: cleanHTML }} />; }

Vue.js XSS Protection

Vue.js provides built-in XSS protection with v-html directive:

// Vue.js template <template> <!-- SAFE - Text interpolation --> <div>{{ userContent }}</div> <!-- DANGEROUS - v-html without sanitization --> <div v-html="userContent"></div> <!-- SAFE - With sanitization --> <div v-html="sanitizedContent"></div> </template> // Vue.js script import DOMPurify from 'dompurify'; export default { data() { return { userContent: 'user input' }; }, computed: { sanitizedContent() { return DOMPurify.sanitize(this.userContent, { ALLOWED_TAGS: [], ALLOWED_ATTR: [] }); } } };

Angular XSS Protection

Angular has built-in sanitization for different contexts:

import { Component } from '@angular/core'; import { DomSanitizer, SafeHtml } from '@angular/platform-browser'; @Component({ selector: 'app-root', template: ` <!-- SAFE - Interpolation --> <div>{{ userContent }}</div> <!-- DANGEROUS - innerHTML binding --> <div [innerHTML]="userContent"></div> <!-- SAFE - With sanitization --> <div [innerHTML]="safeContent"></div> ` }) export class AppComponent { userContent = 'user input'; safeContent: SafeHtml; constructor(private sanitizer: DomSanitizer) { this.safeContent = this.sanitizer.bypassSecurityTrustHtml( this.sanitizer.sanitize(1, this.userContent) || '' ); } }

Svelte XSS Protection

Svelte provides different context for safe and unsafe HTML:

// Svelte component <script> import DOMPurify from 'dompurify'; let userContent = 'user input'; function sanitize(html) { return DOMPurify.sanitize(html, { ALLOWED_TAGS: ['b', 'i', 'em', 'strong'], ALLOWED_ATTR: [] }); } </script> <!-- SAFE - Text content --> <div>{userContent}</div> <!-- DANGEROUS - @html directive --> <div>{@html userContent}</div> <!-- SAFE - With sanitization --> <div>{@html sanitize(userContent)}</div>

Advanced Prevention Techniques

1. Subresource Integrity (SRI)

Ensure third-party scripts haven't been tampered with:

<!-- With integrity hash --> <script src="https://cdn.example.com/library.js" integrity="sha384-oqVuAfXRKap7fdgcCY5uykM6+R9GqQ8K/uxy9rx7HNQlGYl1kPzQho1wx4JwY8wC" crossorigin="anonymous"> </script> // Generating integrity hash const crypto = require('crypto'); const fs = require('fs'); const fileBuffer = fs.readFileSync('library.js'); const hash = crypto.createHash('sha384'); hash.update(fileBuffer); const integrity = 'sha384-' + hash.digest('base64');
